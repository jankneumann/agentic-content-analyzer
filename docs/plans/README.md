# Implementation Plans Archive

This directory contains archived implementation plans for features developed in this project.

## Purpose

- **Historical context**: Understand the reasoning behind design decisions
- **Onboarding**: Help new contributors learn from past implementations
- **Pattern reference**: Reuse successful approaches for similar features
- **PR documentation**: Clear link between planning and implementation

## Naming Convention

Plans are named using the format: `YYYY-MM-DD-feature-name.md`

Example: `2025-01-08-revision-chat-panel.md`

## How Plans Get Here

1. During development, plans are created in `.claude/plans/` using Claude Code's plan mode
2. After the feature PR is merged, plans are moved here for archival
3. The PR description should reference the archived plan

## What Plans Include

- Original requirements and goals
- Component architecture decisions
- API design choices
- Implementation steps taken
- Any deviations from the original plan
