## 2025-04-18 - Pre-compile Regex in Markdown Parsing
**Learning:** Python's `re.compile()` cache only holds a limited number of patterns (default 512). When `re.match` or `re.sub` is used inside a tight loop or frequently called function (like `parse_sections` for markdown), the lookup overhead—even if the pattern is in the cache—can be significant. Benchmarks showed a nearly 2x speedup when moving the compilation to the module level.
**Action:** Always pre-compile regular expressions at the module level when they are used within loops or frequently executed text processing pipelines to eliminate compilation and cache-lookup overhead.
