# Contracts — cloud-db-source-of-truth

Machine-readable interface definitions for the new endpoints and schema changes introduced by this proposal. Consumers (e.g. `~/Coding/agentic-assistant`) can generate typed clients from these artifacts before server implementation completes.

## Files

- `openapi/v1.yaml` — OpenAPI 3.1.0 spec for new `/api/v1/*` endpoints
- `db/schema.sql` — PostgreSQL DDL for new `audit_log` table (aligned with Alembic migration)
- `db/seed.sql` — Test fixture rows for contract tests
- `generated/models.py` — Pydantic models generated from OpenAPI (scaffold; regenerated by `datamodel-code-generator`)
- `generated/types.ts` — TypeScript interfaces generated from OpenAPI (scaffold; regenerated by `openapi-typescript`)

## Regeneration Commands

```bash
# Pydantic models
uvx datamodel-code-generator \
  --input contracts/openapi/v1.yaml \
  --output contracts/generated/models.py \
  --output-model-type pydantic_v2.BaseModel

# TypeScript types
npx openapi-typescript contracts/openapi/v1.yaml \
  --output contracts/generated/types.ts
```

## Drift Check

At integration (wp-integration), a test in `tests/contract/test_openapi_drift.py` compares FastAPI's runtime `/openapi.json` against `openapi/v1.yaml`. Drift fails the test — update both or neither.
